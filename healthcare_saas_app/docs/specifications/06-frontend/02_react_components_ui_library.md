# Prompt: React Components and UI System

## Prompt
Build a reusable component system in `frontend/` optimized for healthcare workflows.

### Requirements
1. Create shared components for forms, tables, filters, status badges, and alerts.
2. Build domain components for:
   - appointment scheduling and timeline,
   - patient profile and records overview,
   - doctor availability management,
   - admin dashboard metrics.
3. Ensure accessibility basics (semantic HTML, labels, keyboard navigation).
4. Keep visual language consistent across roles.

### Deliverables
- Component catalog in `frontend/src/components/`.
- Shared style tokens and theme primitives in `frontend/src/styles/`.
- Story/demo page or docs in `frontend/docs/ui-library.md`.

### Acceptance Criteria
- Components are typed and reusable.
- Forms have consistent validation and error messaging patterns.
- UI can scale without copy-paste component duplication.
